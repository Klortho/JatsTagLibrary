rationale

Rationale

Identifies why this milestone exists, what problem it is solving, and/ or what object cannot be modeled using the regular hierarchy.

Remarks

Best Practice: Although not required by the Tag Set, this should be treated as a required attribute and a value always provided when milestones are used.

Attribute Values

In Elements

<milestone-end>
<milestone-start>
ValueMeaning
Text, numbers, or special charactersThe reason for a milestone or information about an object that cannot be modeled using the regular hierarchy.
Restriction: This is an optional attribute; there is no default.
