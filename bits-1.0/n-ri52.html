<xi:fallback>

XInclude Fallback

To quote the XInclude Recommendation, â€œthe <xi:fallback> element appears as a child of an <xi:include> element. It provides a mechanism for recovering from missing resources. When a resource error is encountered, the <xi:include> element is replaced with the contents of the <xi:fallback> element. If the <xi:fallback> element is empty, the <xi:include> element is removed from the result.

Remarks

Since it is not the business of this Tag Set to specify processing, the <xi:fallback> element will default to EMPTY, but the parameter entity %fallback-model; can be redefined in the customization of any user organization that desires a different fallback result.

Attribute

xmlns:xi XInclude Namespace Declaration

Content Model

<!ELEMENT  xi:fallback  %fallback-model;                             >

Expanded Content Model

(#PCDATA)

Description

Text, numbers, or special characters

This element may be contained in:

<xi:include>
